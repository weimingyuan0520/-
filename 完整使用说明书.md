## 一、系统概述

### 1.1 功能简介

本系统通过光敏传感器实时采集环境光照强度，根据四级可配置阈值自动控制蜂鸣器发出告警，支持 LCD 液晶屏显示、三按键交互、串口 JSON 数据通信、WiFi TCP 网络通信，并配套 Python PC 上位机软件。

### 1.2 核心功能模块

| 模块 | 功能 |
|------|------|
| 传感器采集 | 光敏电阻模拟量 ADC 采集 + 8 阶滑动平均数字滤波 |
| 智能控制 | 5 级光照判定 + 蜂鸣器分级响应 (常响/快闪/关闭/慢闪/中闪) |
| 人机交互 | 3 页面 LCD 显示 + 3 按键 (短按/长按) |
| 串口通信 | JSON 格式上行数据 (500ms 周期) + 下行指令解析 |
| 网络通信 | WiFi STA 模式 + TCP Server (端口 8080) |
| PC 上位机 | 实时数据大字体 + 动态曲线图 + 阈值控制 + 日志保存 |

### 1.3 光照等级与蜂鸣器响应

| 等级 | 名称 | 判定条件 (默认) | 蜂鸣器 |
|------|------|----------------|--------|
| L0 | VERY_BRIGHT 极强光 | > 2000 mV | 持续鸣响 |
| L1 | BRIGHT 偏亮 | 1500 ~ 2000 mV | 持续鸣响 |
| L2 | NORMAL 正常 | 800 ~ 1500 mV | 关闭 |
| L3 | DIM 偏暗 | 350 ~ 800 mV | 慢闪 (0.33Hz) |
| L4 | VERY_DARK 极暗 | < 350 mV | 中速闪 (1Hz) |

> 阈值可通过按键、串口指令、TCP 指令、PC 上位机四种方式实时修改。

### 2.3 接线步骤

#### 第一步：最小系统验证
1. ESP32-S3 开发板通过 USB 连接电脑
2. 烧录程序，观察板载红色 LED (GPIO2) 是否亮起
3. 开机时蜂鸣器自检两声

#### 第二步：LCD 显示屏
```
LCD 引脚    →  ESP32 引脚
GND        →  GND
VCC        →  3V3
SCL (SCK)  →  GPIO21
SDI (MOSI) →  GPIO47
SDO (MISO) →  GPIO48 (可选)
CS         →  GPIO42
RS (DC)    →  GPIO41
RST        →  EN
BL         →  GPIO14
```

#### 第三步：传感器 + 蜂鸣器
```
光敏传感器:
  VCC → 3V3
  GND → GND
  AO  → GPIO5

蜂鸣器 (经 S8050 驱动):
  GPIO6 → 1kΩ → 三极管 B (基极)
  3V3 → 蜂鸣器正极
  蜂鸣器负极 → 三极管 C (集电极)
  三极管 E (发射极) → GND
```

#### 第四步：按键
```
K1 (模式切换) → GPIO15 → 按钮 → GND
K2 (阈值减小) → GPIO17 → 按钮 → GND
K3 (阈值增大) → GPIO8  → 按钮 → GND
```

#### 第五步：串口通信
```
USB 转串口模块 RX  → GPIO36 (ESP32 TX)
USB 转串口模块 TX  → GPIO38 (ESP32 RX) [本板可能未连通]
USB 转串口模块 GND → GND
```

### 2.4 蜂鸣器驱动电路

```
        3V3
         │
     ┌───┴───┐
     │蜂鸣器  │
     └───┬───┘
         │
         C (集电极)
    GPIO6 ──[1kΩ]── B (基极)   S8050 NPN
                    E (发射极)
                      │
                     GND
```

## 四、设备端操作指南

### 4.1 开机

USB 供电后：
1. 板载红色 LED 亮起
2. 蜂鸣器两声自检 (各 100ms)
3. LCD 显示 "Light Monitor" 标题
4. 约 3 秒后 WiFi 连接 (LCD P2 页面可查看状态)
5. 约 5 秒后滤波器就绪，开始正常监控

### 4.2 按键功能

| 按键 | 操作 | 功能 |
|------|------|------|
| **K1** | 短按 | 切换 LCD 页面 (P0 数据 → P1 阈值 → P2 网络 → P0) |
| **K1** | 长按 2 秒 | 快速跳转: 非 P1 → 跳 P1 阈值页 / P1 → 跳 P0 数据页 |
| **K2** | 短按 (P1阈值页) | 当前阈值 -50 |
| **K2** | 长按 (P1阈值页) | 当前阈值 -200 (快速调整) |
| **K2** | 短按 (P0/P2页面) | 强制关闭蜂鸣器 / 恢复自动 |
| **K3** | 短按 (P1阈值页) | 当前阈值 +50 |
| **K3** | 长按 (P1阈值页) | 当前阈值 +200 (快速调整) |
| **K3** | 短按 (P0/P2页面) | 蜂鸣器测试开关 (强制开/恢复自动) |

> 💡 **设计变更**：阈值调整不再依赖"长按 K1 进入配置模式"，改为页面驱动——切到 P1 Thresholds 页面即可直接用 K2/K3 调节，切回其他页面后 K3 恢复蜂鸣器切换功能，互不冲突。

### 4.3 LCD 页面说明

#### P0 — 实时数据页

│  Light Monitor   │ 蓝底标题
├──────────────────┤

├──────────────────┤
│ --- Status ---   │ 分隔线
│ Mode: auto       │ 蜂鸣器工作模式

### 4.4 阈值自助校准

**方法一：按键校准**
1. 短按 K1 切换到 P1 Thresholds 页面
2. 短按 K2/K3 微调 (±50)，长按 K2/K3 快调 (±200)
3. 短按 K1 切回 P0，新阈值立即生效

**方法二：PC 上位机**
在 PC 上位机右侧拖动滑块 → 点击"下发阈值到设备"（需先建立 TCP 网络连接）

**方法三：串口指令**
通过串口助手发送 JSON 指令（注意：本板串口下行需 TCP 替代）

**方法四：TCP 指令**
通过网络调试助手连接 8080 端口后发送 JSON 指令

### 4.5 校准参考

| 现象 | 原因 | 调整方法 |
|------|------|---------|
| 室内正常光下蜂鸣器响 | NORMAL 范围太窄 | 增大 TH_MID_H |
| 强光照射无反应 | BRIGHT 阈值太高 | 减小 TH_MID_H |
| 遮挡传感器无反应 | DIM/VERY_DARK 阈值太低 | 增大 TH_LOW / TH_MID_L |

### 5.4 操作流程

1. **连接设备**
   - **推荐方式 (TCP)**: 输入 ESP32 IP 地址 (查看 LCD P2 页) → 端口 8080 → 点击"连接网络"（双向通信，可下发指令）
   - 串口方式: 左侧下拉选择 COM4 → 点击"连接串口"（仅接收数据，本板串口下行硬件不通）

2. **查看数据**
   - 中间大字体实时刷新光敏值和等级
   - 曲线图自动滚动 60 秒历史
   - 4 条彩色虚线标注当前阈值位置

3. **控制设备**
   - 蜂鸣器: 三个按钮手动开/关/自动
   - 阈值: 拖动滑块 → 点击"下发阈值到设备"
   - 查询: 点击"查询设备状态"

4. **日志管理**
   - 底部日志区自动记录所有收发数据
   - "清空" 清除当前日志
   - "保存 TXT" 保存为文本文件
   - "保存 CSV" 保存为表格文件 (可用 Excel 打开)

## 六、通信协议详解

### 6.1 串口参数

| 参数 | 值 |
|------|-----|
| 波特率 | 115200 bps |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验 | 无 |
| 流控 | 无 |

### 6.2 上行 JSON (设备 → PC，每 500ms)

```json
{
  "type": "data",
  "light": 985,
  "light_level": 2,
  "light_level_name": "NORMAL",
  "buzzer": "off",
  "buzzer_mode": "auto",
  "th_high": 2000,
  "th_mid_h": 1500,
  "th_mid_l": 800,
  "th_low": 350,
  "wifi": "connected",
  "ip": "10.109.232.90",
  "page": 0,
  "ts": 12500
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| `type` | string | 固定 `"data"` |
| `light` | int | 当前光敏值 (mV, 0~3600, 反转后) |
| `light_level` | int | 等级编号 (0~4) |
| `light_level_name` | string | 等级名称 |
| `buzzer` | string | `"on"` / `"off"` |
| `buzzer_mode` | string | `"auto"` / `"on"` / `"off"` |
| `th_high` | int | 极强光阈值 |
| `th_mid_h` | int | 偏亮阈值 |
| `th_mid_l` | int | 偏暗阈值 |
| `th_low` | int | 极暗阈值 |
| `wifi` | string | `"connected"` / `"disconnected"` |
| `ip` | string | IP 地址 |
| `page` | int | LCD 页面 (0~2) |
| `ts` | int | 系统运行毫秒数 |

### 6.3 下行指令 (PC → 设备)

**设置阈值:**
```json
{"cmd":"set_threshold","th_high":2000,"th_mid_h":1500,"th_mid_l":800,"th_low":350}
```
> 约束: th_high > th_mid_h > th_mid_l > th_low

**蜂鸣器控制:**
```json
{"cmd":"buzzer_ctrl","action":"on"}    // 强制开启
{"cmd":"buzzer_ctrl","action":"off"}   // 强制关闭
{"cmd":"buzzer_ctrl","action":"auto"}  // 恢复自动模式
```

**查询状态:**
```json
{"cmd":"get_status"}
```
> 响应: 返回完整上行 JSON 数据

### 6.4 TCP 网络通信

| 参数 | 值 |
|------|-----|
| 协议 | TCP |
| 模式 | Server |
| 端口 | 8080 |
| 数据格式 | JSON (与串口协议完全相同，兼容带空格/无空格两种格式) |
| 换行符 | `\r\n` |

> 💡 JSON 指令兼容 `{"cmd":"set_threshold",...}` 和 `{"cmd": "set_threshold", ...}` 两种格式。

## 七、通信验证操作指南

### 7.1 串口调试助手验证

**工具**: `串口调试助手\sscom33.exe`

#### 配置串口

| 参数 | 值 |
|------|-----|
| 端口号 | COM6 |
| 波特率 | 115200 |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | None |
| 流控 | 无 |

在软件右下角勾选 **"发送新行"** (自动追加 `\r\n`)。

#### 操作步骤

1. 打开 `sscom33.exe`
2. 按上表配置串口参数
3. 点击 **"打开串口"** 按钮
4. 等待约 12 秒 (ESP32 启动 + WiFi 连接 + 滤波预热)
5. 观察接收区应每 500ms 刷新一条 JSON 数据

#### 上行验证 (设备 → PC)

接收区应看到类似数据：

```
{"type":"data","light":985,"light_level":2,"light_level_name":"NORMAL","buzzer":"off","buzzer_mode":"auto","th_high":2000,"th_mid_h":1500,"th_mid_l":800,"th_low":350,"wifi":"connected","ip":"10.109.232.90","page":0,"ts":12500}
```

✅ 收到此数据即证明**上行通信正常**。

#### 下行验证 (PC → 设备)

在发送区依次输入并发送以下指令：

**指令 1 — 查询设备状态:**
```
{"cmd":"get_status"}
```
> 预期: 返回完整 JSON 状态数据

**指令 2 — 强制打开蜂鸣器:**
```
{"cmd":"buzzer_ctrl","action":"on"}
```
> 预期: 蜂鸣器持续响，LCD 显示 `Mode: on`

**指令 3 — 强制关闭蜂鸣器:**
```
{"cmd":"buzzer_ctrl","action":"off"}
```
> 预期: 蜂鸣器停止，LCD 显示 `Mode: off`

**指令 4 — 恢复自动模式:**
```
{"cmd":"buzzer_ctrl","action":"auto"}
```
> 预期: 蜂鸣器恢复自动阈值控制

**指令 5 — 修改阈值:**
```
{"cmd":"set_threshold","th_high":2200,"th_mid_h":1600,"th_mid_l":900,"th_low":400}
```
> 预期: LCD P1 页面显示新阈值，蜂鸣器按新阈值响应

> ⚠️ **注意**: 本开发板的 CH340 TX (PC发→ESP32收) 方向可能未接通 GPIO38。若发送指令后无响应，请改用 [7.2 网络调试助手验证](#72-网络调试助手验证)。

### 7.2 网络调试助手验证

**工具**: `网络调试助手\NetAssist.exe`

#### 配置连接

| 参数 | 值 |
|------|-----|
| 协议类型 | **TCP Client** |
| 远程主机地址 | **查看 LCD P2 页显示的 IP** (如 10.109.232.90) |
| 远程主机端口 | **8080** |

#### 操作步骤

1. 打开 `NetAssist.exe`
2. 左侧协议类型选择 **"TCP Client"**
3. 输入远程主机地址 (查看 LCD 网络页 P2)
4. 输入远程主机端口 **8080**
5. 点击 **"连接"** 按钮
6. 连接成功后接收区每 500ms 刷新 JSON 数据
7. 在发送区输入指令，点击 **"发送"**

#### 上行验证 (设备 → PC)

连接成功后，接收区自动显示 JSON 数据：

```
{"type":"data","light":985,"light_level":2,...,"wifi":"connected","ip":"10.109.232.90",...}
```

✅ 收到此数据即证明**网络上行通信正常**。

#### 下行验证 (PC → 设备)

在发送区依次发送以下指令，观察蜂鸣器和 LCD 响应：

**测试 A — 蜂鸣器控制流程:**
```
发送: {"cmd":"buzzer_ctrl","action":"on"}
→ 蜂鸣器应持续响 ✗
发送: {"cmd":"buzzer_ctrl","action":"off"}
→ 蜂鸣器应停止 ✓
发送: {"cmd":"buzzer_ctrl","action":"auto"}
→ 恢复自动模式 (根据光照自动控制)
```

**测试 B — 阈值修改:**
```
发送: {"cmd":"set_threshold","th_high":1800,"th_mid_h":1200,"th_mid_l":700,"th_low":300}
→ LCD P1 页阈值数值应变为 1800/1200/700/300
→ 用手电照射传感器，因阈值降低，更容易触发 BRIGHT/VERY_BRIGHT
```

**测试 C — 恢复默认阈值:**
```
发送: {"cmd":"set_threshold","th_high":2000,"th_mid_h":1500,"th_mid_l":800,"th_low":350}
→ LCD P1 页阈值恢复默认值
```

**测试 D — 完整功能验证流程:**
```
1. 查询: {"cmd":"get_status"}
2. 强开蜂鸣: {"cmd":"buzzer_ctrl","action":"on"}
3. 确认蜂鸣器响了，再恢复: {"cmd":"buzzer_ctrl","action":"auto"}
4. 修改阈值为低值: {"cmd":"set_threshold","th_high":1200,"th_mid_h":900,"th_mid_l":500,"th_low":200}
5. 用物体遮光，确认蜂鸣器自动响应
6. 恢复默认: {"cmd":"set_threshold","th_high":2000,"th_mid_h":1500,"th_mid_l":800,"th_low":350}
```

#### 串口与网络双通道对比

| 通道 | 上行 (设备→PC) | 下行 (PC→设备) | 推荐用途 |
|------|---------------|---------------|---------|
| 串口 (COM4, 115200) | ✅ 可用 | ❌ 本板不通 | 查看数据流、调试 |
| 网络 (TCP :8080) | ✅ 可用 | ✅ 可用 | 下发指令、上位机通信 |

> 如果 LCD P2 显示的 IP 不是 `10.109.232.90`，请以 LCD 实际显示的 IP 为准。

## 八、故障排查

### 8.1 常见问题

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 板载 LED 不亮 | 烧录失败 / 供电不足 | 重新烧录，检查 USB 线 |
| LCD 白屏 | SPI 接线错误 | 检查 GPIO21/47/42/41 |
| LCD 黑屏无背光 | BL 引脚未拉高 | 检查 GPIO14 |
| LCD 文字乱码/闪烁 | 多任务冲突 / 字符串超 15 列 | 已修复 v2.0+ |
| 光敏值始终为 0 | 传感器未接 | 检查 GPIO5 电压 |
| 光敏值不随光照变化 | 传感器电路问题 | 检查分压电阻 |
| 光照强但数值减小 | 正常现象 (ADC 已反转) | 无需处理 |
| 蜂鸣器始终不响 | 阈值过高 / 驱动电路问题 | 听开机自检声判断 |
| 开机自检有声，正常不响 | 光照未超阈值 | 降低 TH_MID_H 或用手电测试 |
| 按键无反应 | 按键接触不良 | 测 GPIO 按下时是否为 0V |
| WiFi 连不上 | SSID/密码错误 | 修改 `wifi-sta.h` |
| 串口无数据 | TX 线接错 / 波特率不对 | 确认 GPIO36, 115200 |
| TCP 连不上 | IP 变化 / 端口被占用 | 查看 LCD P2 当前 IP |
| PC 上位机无法连接串口 | 串口被其他程序占用 | 关闭串口助手 / 另一 Python 进程 |
| PC 上位机中文显示方框 | 缺少中文字体 | 安装 Microsoft YaHei 字体 |

### 8.2 蜂鸣器自检判断

开机时蜂鸣器会响两短声 (各 100ms):

- ✅ **听到两声** → 蜂鸣器硬件和驱动正常
- ❌ **完全无声** → 检查蜂鸣器电路: GPIO6→1kΩ→S8050 B 极, C 极→蜂鸣器→3V3, E 极→GND

### 8.3 串口下行不通的替代方案

本开发板的 CH340 TX 方向未正确连通 ESP32 GPIO38，导致串口下行指令无响应。替代方案:
1. **TCP 网络下发**: 连接 WiFi 后通过 TCP 8080 端口发送指令
2. **按键本地修改**: 长按 K1 进入配置模式，K2/K3 调节阈值
3. **PC 上位机 TCP 连接**: 输入 ESP32 IP → 连接网络 → 使用控制面板下发

## 附录 A: 阈值换算参考

光敏传感器 LDR 分压电路 (3.3V → 10kΩ → GPIO5 → LDR → GND):

| 光照条件 | LDR 电阻 (约) | 原始 ADC (mV) | 反转后 (mV) | 等级 |
|---------|-------------|-------------|-----------|------|
| 完全遮光 | ~100kΩ | ~3000 | ~600 | DIM~VERY_DARK |
| 室内正常光 | ~30kΩ | ~2600 | ~1000 | NORMAL |
| 手机闪光灯 | ~5kΩ | ~1100 | ~2500 | VERY_BRIGHT |
| 阳光直射 | ~1kΩ | ~300 | ~3300 | VERY_BRIGHT |

> 实际值因 LDR 型号、分压电阻值、环境光而异，需现场校准。
