# 智能光敏监控系统 — 使用说明书

> **主控芯片**: ESP32-S3 (普中 AI 机器视觉开发板)  
> **版本**: v2.0  
> **日期**: 2026-06-13

---

## 一、系统概述

本系统通过光敏传感器实时采集环境光照强度，根据多级阈值自动控制蜂鸣器发出不同频率的告警，支持 LCD 液晶屏显示、按键交互、串口通信、WiFi 网络通信四大交互方式。

### 核心功能
- 光敏传感器模拟量采集 + 滑动平均数字滤波
- 5 级光照智能判定 (VERY_BRIGHT / BRIGHT / NORMAL / DIM / VERY_DARK)
- 蜂鸣器 4 种响应模式 (持续 / 快速 / 中速 / 慢速)
- 3 页面 LCD 液晶显示 (数据页 / 阈值页 / 网络页)
- 按键交互 (页面切换 / 阈值本地修改)
- 串口 JSON 数据上行 + 指令下行
- WiFi TCP Server 网络通信

## 二、硬件连接速查

| 外设 | ESP32-S3 引脚 | 说明 |
|------|-------------|------|
| 光敏传感器 (模拟) | **GPIO5** (ADC1_CH4) | 接光敏模块 AO 输出 |
| 光敏传感器 VCC | 3V3 | 模块供电 |
| 光敏传感器 GND | GND | 共地 |
| 蜂鸣器 | **GPIO6** | 通过三极管驱动 |
| LED (板载红色) | **GPIO2** | 开发板自带 |
| 按键 K1 | **GPIO15** | 页面切换/配置 |
| 按键 K2 | **GPIO17** | 阈值减小 |
| 按键 K3 | **GPIO8** | 阈值增大 |
| LCD 显示屏 | GPIO21/47/48/42/41/14 | SPI 接口 ST7789 |
| 串口 TX | **GPIO36** | 接 USB 串口模块 RX |
| 串口 RX | **GPIO38** | 接 USB 串口模块 TX |

## 三、按键操作

| 按键 | 操作 | 功能 |
|------|------|------|
| **K1** | 短按 | 切换 LCD 页面 (P0→P1→P2→P0 循环) |
| **K1** | 长按 2 秒 | 进入/退出阈值配置模式 |
| **K2** | 短按 (配置模式) | 当前阈值 -50 |
| **K2** | 长按 (配置模式) | 当前阈值 -200 (快速调整) |
| **K3** | 短按 (配置模式) | 当前阈值 +50 |
| **K3** | 长按 (配置模式) | 当前阈值 +200 (快速调整) |

## 五、光照等级与蜂鸣器响应

| 等级 | 判定条件 | 蜂鸣器模式 | 说明 |
|------|---------|-----------|------|
| **VERY_BRIGHT** | > TH_HIGH (2700mV) | 持续鸣响 | 极强光告警 |
| **BRIGHT** | TH_MID_H ~ TH_HIGH | 0.5Hz 慢闪 (1s on/off) | 偏亮提醒 |
| **NORMAL** | TH_MID_L ~ TH_MID_H | 关闭 | 正常范围 |
| **DIM** | TH_LOW ~ TH_MID_L | 0.33Hz 极慢 (1.5s on/off) | 偏暗提醒 |
| **VERY_DARK** | < TH_LOW (400mV) | 1Hz 中速 (500ms on/off) | 极暗告警 |

> **注意**: 光敏传感器使用 LDR 分压电路，**光照越强 ADC 值越大**（已内部反转处理）。

## 六、串口通信协议

**参数**: 115200bps / 8N1

### 上行 (设备 → PC) — 每 500ms 自动发送
```json
{
  "type": "data",
  "light": 998,
  "light_level": 2,
  "light_level_name": "NORMAL",
  "buzzer": "off",
  "buzzer_mode": "auto",
  "th_high": 2700,
  "th_mid_h": 2000,
  "th_mid_l": 900,
  "th_low": 400,
  "wifi": "connected",
  "ip": "10.109.232.90",
  "page": 0,
  "ts": 12500
}
```

### 下行 (PC → 设备)

**修改阈值**:
```json
{"cmd":"set_threshold","th_high":3000,"th_mid_h":2200,"th_mid_l":1000,"th_low":500}
```

**控制蜂鸣器**:
```json
{"cmd":"buzzer_ctrl","action":"on"}    // 强制开启
{"cmd":"buzzer_ctrl","action":"off"}   // 强制关闭
{"cmd":"buzzer_ctrl","action":"auto"}  // 恢复自动模式
```

**查询状态**:
```json
{"cmd":"get_status"}
```

## 七、WiFi 网络通信

| 参数 | 值 |
|------|-----|
| 模式 | WiFi STA (客户端) |
| SSID | `666` |
| 密码 | `15731491213` |
| 协议 | TCP Server |
| 端口 | **8080** |
| 数据格式 | JSON (同串口协议) |

### 修改 WiFi 配置

编辑 `components\wifi-sta\wifi-sta.h`:
```c
#define DEFAULT_SSID    "你的WiFi名"
#define DEFAULT_PWD     "你的WiFi密码"
```

### 使用网络调试助手

1. 打开 `网络调试助手\NetAssist.exe`
2. 协议类型: TCP Client
3. 远程主机地址: LCD P2 页面显示的 IP (如 10.109.232.90)
4. 远程主机端口: 8080
5. 点击 "连接"
6. 可接收 JSON 数据/发送控制指令

## 八、阈值自助校准

### 方法一：按键校准
1. **长按 K1** 2 秒 → 进入配置模式 (LCD 显示 "** CONFIG MODE")
2. **短按 K2/K3** → 微调阈值 ±50
3. **长按 K2/K3** → 快速调整 ±200
4. **短按 K1** → 退出配置模式

### 方法二：串口指令
通过 `串口调试助手\sscom33.exe` 发送 JSON 阈值指令

### 方法三：网络指令
通过网络调试助手 TCP 连接后发送 JSON 指令

### 校准参考
| 光照条件 | 期望判定 | 建议操作 |
|---------|---------|---------|
| 室内正常光下蜂鸣器响 | 应为 NORMAL | 增大 TH_MID_H |
| 强光照射无反应 | 应为 BRIGHT/VERY_BRIGHT | 减小 TH_HIGH/TH_MID_H |
| 遮挡传感器无反应 | 应为 DIM/VERY_DARK | 增大 TH_LOW/TH_MID_L |

## 十、故障排查

| 现象 | 可能原因 | 解决方法 |
|------|---------|---------|
| 板载 LED 不亮 | 烧录失败/供电不足 | 重新烧录，检查 USB 线 |
| LCD 白屏 | SPI 接线错误 | 检查 GPIO21/47/42/41 接线 |
| LCD 黑屏/无背光 | BL 引脚未拉高 | 检查 GPIO14 |
| 光敏值始终为 0 | 传感器未接/ADC 故障 | 测 GPIO5 电压 |
| 蜂鸣器持续响 | 光敏值高于 TH_HIGH | 进入配置模式调高 TH_HIGH |
| 按键无反应 | 按键接触不良/内部上拉失效 | 测 GPIO 电压，按下应为 0V |
| WiFi 连不上 | SSID/密码错误 | 修改 wifi-sta.h 重新编译 |
| TCP 连不上 | IP 地址变化 | 查看 LCD P2 页获取当前 IP |
| 串口无数据 | 波特率不匹配/TX 接线错 | 确认 115200bps，检查 GPIO36 |
